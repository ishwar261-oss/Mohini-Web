🔄 Code Flow

How MOHINI
Processes Your Input

From keystroke to AI response — watch the full data pipeline.

01
⌨️

User Input

User types a message or slash command in the ChatPanel. Input is captured by the Swing text field's action listener.

ChatPanel.java
sendButton.addActionListener(e -> {
  String input = inputField.getText().trim();
  if (!input.isEmpty()) processInput(input);
});
02
🔍

Validation & Routing

InputValidator sanitizes the input. CommandHandler checks if it starts with / and routes to the correct command class.

CommandHandler.java
if (input.startsWith("/")) {
  Command cmd = registry.get(input.split(" ")[0]);
  return cmd != null ? cmd.execute(args) : "Unknown command";
} else {
  return aiService.chat(input); // → AI path
}
Branching Logic
🤖

AI Chat Path

OpenAIService.java builds the JSON payload with message history, calls OpenRouter REST API, and returns the streamed response.

POST api.openrouter.ai/api/v1/chat
// model: openai/gpt-4o-mini
// Authorization: Bearer $API_KEY
// messages: [{role, content}...]
OR
⚡

Command Path

Matched command (e.g. CalcCommand) runs locally: parses args, executes logic, and returns plain text — no API call needed.

class CalcCommand implements Command {
  public String execute(String args) {
    return Evaluator.eval(args);
  }
}
04
💾

Storage Layer

Depending on the command, TodoManager, NotesManager, or ChatHistoryManager persist data as JSON files in the data/ folder.

TodoManager.java
public void saveTodos(List<Todo> todos) {
  JSONArray arr = new JSONArray(todos);
  Files.writeString(Path.of("data/todos.json"), arr.toString());
}
05
🎨

UI Rendering

The response is rendered in ChatUI as a styled message bubble. TTSEngine optionally reads it aloud using Windows PowerShell Speech.

TTSEngine.java
Runtime.getRuntime().exec(new String[]{
  "powershell", "-Command",
  "Add-Type -AssemblyName System.Speech; ..."
  + "$s.Speak('" + text + "')"
});
